联结词（5个常用）

蕴含： 与因果关系无关 只反映真值关系 \[ 令 P:“陈老师讲课.” Q:“他来听课.” 则“只有陈老师讲课他才来听课”应译为\\ \mathcal {Q \rightarrow P. \\ 或\neg (Q\land \neg P) = \neg Q \lor P = Q \rightarrow P} \] P是Q的充分条件

Q是P的必要条件

若P, 则Q

除非Q, 才P

只要P,就Q

只有Q, 才P

Q每当P

P仅当Q

除非Q，否则非P

命题的解释：对P、Q的每组真值组合(真值指派)，都称作命题A的一个 解释

1.3 合式公式(Well Formed Formula)（或称命题公式）

定义(命题公式):

(1)命题常量和命题变量是命题公式.

(2)如果A、B是命题公式,则(A), (AB), (AB), (AB)和(AB)也是命题公式.

(3)所有命题公式都通过有限次的使用(1)(2)得到.

约定：

为简化括号的使用,可规定联结词的优先级

 由高到低: \[ \neg \land \lor \rightarrow \leftrightarrow \]  多个同一连接词:自左到右次序

 最外层括号常省略

1.4

重言式是可满足的

以下关系是显然的

  1. A永真 iff 非A永假.

  2. A可满足 iff 非A非永真.

  3. A非可满足 iff A永假.

  4. A非永假 iff A可满足.

带入规则：

A是一个公式，对A使用代入规则得公式B， 若A是重言式，则B也是重言式

使用代入规则证明重言式

异或： \[ A \oplus B = (\neg A \land B) \lor(A\land \neg B) \]

类型推导：

在传统 C 和 C++中，参数的类型都必须明确定义，这其实对我们快速进行编码没有任何帮助，尤其是当我们面对一大堆复杂的模板类型时，必须明确的指出变量的类型才能进行后续的编码，这不仅拖慢我们的开发效率，也让代码变得又臭又长。

（1）nullptr

nullptr 出现的目的是为了替代 NULL。在某种意义上来说，传统 C++ 会把 NULL、0 视为同一种东西，这取决于编译器如何定义 NULL，有些编译器会将 NULL 定义为 ((void*)0)，有些则会直接将其定义为 0。

C++ 不允许直接将 void * 隐式转换到其他类型，但如果 NULL 被定义为 ((void*)0)，那么当编译 char *ch = NULL; 时，NULL 只好被定义为 0。而这依然会产生问题，将导致了 C++ 中重载特性会发生混乱，考虑：
